# Bank Customer Analysis

SQL project focused on building a customer-level dataset from a relational banking database.

The objective is to transform data distributed across multiple tables (customers, accounts, and transactions) into a single integrated structure, supporting behavioral analysis and data-driven decision-making.

---

## Objective

Build a denormalized customer-level table that enables:
- analysis of customer financial behavior;
- comparison of different customer profiles on a consistent basis;
- support for segmentation activities;
- preparation of data for predictive analytics.

---

## Data

The project is based on a predefined relational database structure, composed of multiple connected tables (customers, accounts, transaction types, and transactions).

The original database is not included in this repository.

---

## Tools

- SQL (language)
- MySQL (MySQL Workbench)

---

## Results

The project resulted in the creation of a denormalized customer-level table integrating:
- customer demographic information;
- account-related data;
- transaction-based indicators.

The resulting structure provides a clear and synthetic view of customer behavior and is ready to be used for advanced analysis and machine learning models.

---

## SQL Script

The `.sql` file contains the full process used to build the final table, including:
- initial table inspection;
- creation of temporary tables;
- calculation of indicators;
- final data integration.

The final table is not stored as a static file, as it is generated by executing the SQL script.

The script is designed to work on a predefined database structure, which is not included in this repository.

Note: The SQL project is written in Italian.
